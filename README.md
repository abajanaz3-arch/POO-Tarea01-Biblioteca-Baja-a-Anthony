<p>Estructura del Caso de Estudio

    1. Introducción

Contexto:  "Sistema de Gestión de Biblioteca UNEMI".

Objetivo: Modificar el sistema existente para que permita gestionar préstamos tanto a estudiantes como a profesores, aplicando principios fundamentales de la Programación Orientada a Objetos (POO).

     2. Análisis del Problema

Situación Inicial: El sistema original estaba acoplado  a la clase Estudiante, lo que impedía prestar libros a otros tipos de usuarios sin duplicar el código.

Solución Propuesta: Actualizar el código utilizando Herencia para crear un sistema más flexible y escalable.



     3. Implementación

Un resumen de las modificaciones a nivel de código. Aquí puedes mencionar los archivos clave que trabajamos:

Creación de modelo/profesor.py.

Actualización de modelo/prestamo.py (cambio de estudiante a persona).

Actualización de modelo/biblioteca.py (cambio de listas y métodos de búsqueda para aceptar Persona).

     4. Pruebas y Resultados (Ejecución)

Escenarios validados: Explicar lo que hace el main.py (Registro de usuarios, préstamos exitosos a distintos roles, validación de libros no disponibles, devoluciones).



    5. Conclusiones

Beneficios del rediseño:  Resaltar que el código ahora es escalable y mantenible.
